User Stories:
1. As a casual consumer, I want to filter movies that other people have watched by highest rating, so I can easily find recommendations for movies to watch next.
2. As a frequent movie watcher, I want to create an account, so I can log information about the movies I've watched.
3. As someone who has friends who use the app, I want to friend their accounts, so I can see their watch history and favorite genres.
4. As a registered user, I want to rate a movie or TV show, so that I can contribute to its overall ranking and track my opinions.
5. As a registered user, I want to write a review for a movie or TV show, so that I can share my thoughts with other users.
6. As a user, I want to save my watch history, so that I can keep track of what I’ve already watched.
7. As a user, I want to filter movies by specific directors, so I can watch many of their works. 
8. As a registered user, I want to be able to interact with others' reviews such as commenting and liking, so reviews can become popular.
9. As a user, I want to filter movies by genre, so I can find similar shows to those I like.

Exceptions:
1. Exception: Media title does not exist. If a user tries to search for a specific media by title that does not exist in the system, the search will give back a message stating that the title cannot be found and encourage the user to search again.
2. Exception: Duplicate media titles. If a user tries to log a piece of media that shares a title with another, they will be prompted to enter the year of release to differentiate the two entities.
3. Exception: A user wants to hide spoiler content in a review. If a user includes spoilers within a review, they will have the option to add a "contains spoilers" warning, which hides the content of their review from users who have not logged the media.
4. Exception: User tries to create an account with an email that already exists. If a user signs up with an email already associated with another account, the system will return an error and prompt them to log in or use a different email.
5. Exception: Invalid rating input. If a user submits a rating outside the allowed range (e.g., below 1 or above 5), the system will reject the input and ask them to enter a valid rating.
6. Exception: Empty review submission. If a user tries to submit a review without any content, the system will return an error and require them to enter text before posting.
7. Exception: Email address associated with the account is not a real address. If a user enters an email address that is not valid, it will prompt the user to re-enter the address.
8. Exception: User submits a review while not logged in. If a user attempts to submit a review without being logged in, the system will prompt them to the login/signup page.
9. Exception: User tries to edit or delete another user’s review. If a user attempts to change a review they do not own, the system will block the action and display an “Unauthorized action” error message.
